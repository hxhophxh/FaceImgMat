## 📝 文档贡献

欢迎完善文档！请遵循以下规范：

### 文档命名
- 使用小写字母和连字符
- 描述性名称，如 `user-guide.md`
- 避免使用空格和特殊字符

### 文档结构
```markdown
# 文档标题

简短的文档说明（1-2句话）

## 目录
（如果文档较长）

## 主要内容

## 示例代码

## 常见问题

## 相关文档
```

### 格式规范
- 使用 Markdown 格式
- 代码块标注语言类型
- 添加 emoji 提高可读性
- 截图放在 `docs/images/` 目录
